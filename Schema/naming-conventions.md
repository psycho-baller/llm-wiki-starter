# Naming Conventions

Use readable, stable filenames. Prefer lowercase words separated by hyphens.

Examples:

- `Raw/Sources/llm-wiki-starter-demo-source.md`
- `Wiki/Concepts/raw-vs-compiled-knowledge.md`
- `Wiki/Topics/llm-wiki-workflow.md`
- `Wiki/Projects/build-core-llm-wiki.md`
- `Wiki/Logs/initial-demo-ingest.md`

Guidelines:

- Use lowercase snake_case for every machine-readable frontmatter key.
- Keep filenames short enough to scan.
- Avoid dates unless the note is explicitly a log or time-bound artifact.
- Avoid vague names such as `notes.md`, `summary.md`, or `misc.md`.
- Keep source filenames stable after compiled notes cite them.
- Use frontmatter `aliases` for alternate wording instead of duplicating notes.
