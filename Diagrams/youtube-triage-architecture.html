YouTube Triage Architecture

AI-backed filtering pipeline for deciding whether clipped YouTube videos should be skipped, skimmed, watched, processed into Wiki notes, or deferred.

Capture Boundary

  • • Clipper creates factual Raw source metadata only
  • • `decision: pending` keeps judgment out of capture
  • • `consumption_status: unwatched` is the default
  • • Bases can display the pending review queue

Triage Execution

  • • `wiki_tool.py` orchestrates deterministic workflow
  • • `youtube_triage.py` calls Codex, Gemini, Claude, or manual mode
  • • Providers return JSON only; Python writes files

Validated Output

  • • Schema enforces scores, risks, decision, reason, expected gain
  • • Raw source gets frontmatter fields and `## Triage Notes`
  • • Catalog, manifest, and lint gate keep vault state consistent
